# ⚽ 盘口裁决者

> 基于竞彩网官方 API 实时数据，AI 自主拆解盘口信号，裁决每一张票。不靠运气，只靠数据结构。支持赛后复盘，持续学习优化。

**当前版本：V6.0.0**

---

## 这是什么？

一个 Claude Code Skill，用于每日竞彩足球 AI 分析。调用竞彩网官方 API 获取实时固定奖金数据，由 AI 模型自主判断让球结构、奖金分布和赛事走势，输出三套资金分配方案和清晰的购买指令。

**核心原则：只约束格式和规则，不约束判断。** 不同 AI 模型（Claude / DeepSeek / GPT / Gemini / Qwen）面对同一组数据，必须产出不同的方案。

---

## 功能一览

### 📊 赛前分析模式（默认）

| 功能 | 说明 |
|------|------|
| 实时数据抓取 | 调用竞彩网官方 API `getMatchCalculatorV1`，获取当日可售赛事的固定奖金 |
| AI 自主分析 | 基于让球结构、固定奖金分布、单关状态，自主判断方向 |
| 三套方案 | 稳健保底（2串1）+ 价值发现（单关）+ 高危搏杀（比分冷门） |
| 购买指令 | 用球队名 + 大白话说清每张票买什么，不模棱两可 |
| 风控规则 | 2 元法则、资金绝对平衡、终止分支、反幻觉、术语规范 |

### 🔍 赛后复盘模式

| 功能 | 说明 |
|------|------|
| 赛果搜索 | 通过 WebSearch 多源交叉验证获取实际比分 |
| 逐场对比 | 预测方向 vs 实际比分 vs 让球结果，判断正误 |
| 偏差归因 | 盘口漏读？基本面误判？随机事件？逐条分析根因 |
| 教训提炼 | 「如果重来」视角总结可改进点 |
| 学习沉淀 | 复盘教训自动写入 SKILL.md「历史教训」附录，后续分析自动参考 |

### 🤖 每日自动化

| 功能 | 说明 |
|------|------|
| 预测存储 | 每次分析完成后自动保存预测快照到 `predictions/{matchDate}.json` |
| 复盘自动读取 | 复盘时自动读取存储的预测数据，无需手动提供 |
| 定时调度 | 配合 `/loop` 指令可实现早间分析 + 晚间复盘全自动闭环 |
| 持续学习 | 复盘教训写入 SKILL.md，下次分析时 AI 自动参考 |

## 如何调用

### 安装

将本仓库克隆到 Claude Code 的 skills 目录：

```bash
git clone https://github.com/coryeleven/sporttery-advisor-skill.git \
  ~/.claude/skills/sporttery-advisor-skill
```

### 赛前分析

**基本用法**（分析今天可买的赛事）：

```text
预算 M = 50。请按"盘口裁决者"Skill 执行。
```

**指定日期**：

```text
预算 M = 50。分析明天的比赛。
```

```text
预算 M = 100。分析 6 月 20 号的赛事。
```

**参数说明**：
- `M`：预算金额，必须是 ≥6 的偶数（体彩单注 2 元，所以必须是偶数）
- 日期：支持「今天」「明天」「X月X号」，默认当天
- 日期逻辑：业务日 N 买赛事日期 N+1 的票（比如 6 月 19 日买的是 6 月 20 日开赛的球）

### 赛后复盘

**触发方式**（赛果出来后）：

```text
复盘昨天的预测。
```

```text
回顾 6 月 19 号的比赛结果。
```

```text
对比赛果，看看我预测准不准。
```

触发后 Skill 会：
1. 自动读取 `predictions/{matchDate}.json` 中存储的预测数据
2. 搜索当天所有比赛的官方赛果
3. 逐场对比你的预测 vs 实际结果
4. 分析每处偏差的原因
5. 总结教训，写入 SKILL.md「历史教训」

### 每日自动化（可选）

通过 Claude Code 的 `/loop` 指令实现无人值守：

```text
# 早间分析（每天上午 9:00 自动跑）
/loop 9:00 预算 M = 50。按盘口裁决者执行分析。

# 晚间复盘（每天下午 15:00，等球都踢完）
/loop 15:00 复盘今天的比赛。
```

自动化工作链：
```
上午 9:00 → 抓取 API → AI 分析 → 输出方案 → 存储 predictions/日期.json
下午 15:00 → 读取预测 → 搜索赛果 → 逐场对比 → 教训写入 SKILL.md
次日分析时 → AI 自动读「历史教训」→ 避免重复犯错
```

## 方案结构

| 方案 | 标签 | 投入占比 | 玩法 | 风险 |
|------|------|----------|------|------|
| 方案一 | 🟢 稳健保底（必买） | 60%-70% | 让球胜平负 2 串 1 | 低 |
| 方案二 | 🟠 价值发现（可选） | 20%-30% | 胜平负 单关复式 | 中 |
| 方案三 | 🔴 高危搏杀（可不买） | 10%-15% | 猜比分 单挑 | 高 |
| 购买建议 | 🔵 最终购买指令 | 总和 = M | 资金分配 + 出票顺序 | — |

---

## 风控规则

- **2 元法则**：所有金额为正偶数，投注金额 = 2 × 组合数 × 倍数
- **资金绝对平衡**：三套方案金额之和精确等于 M
- **终止分支**：预算非法、API 不可用、无赛事时立即终止
- **反幻觉**：严禁编造任何数据，缺失字段不得补全
- **术语规范**：说"固定奖金"不说"赔率"，说"理论返还"不说"稳赚"

## 技术细节

- **API 接口**：`webapi.sporttery.cn/gateway/uniform/football/getMatchCalculatorV1.qry`
- **数据源**：中国体育彩票竞彩网官方
- **默认输出**：Markdown 纯文本
- **日期规则**：业务日 N 买赛事日期 N+1 的票
- **模型独立性**：不指定具体选场/方向/比分，全部由 AI 自主决策

---

## 风险提示

本项目只做结构化分析和数据报告生成，**不构成投注建议**。所有固定奖金、可售状态、开奖结果以竞彩网官方页面和实际出票时为准。历史复盘数据仅供分析参考，不代表未来表现。
